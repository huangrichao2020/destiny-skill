# 内容化输出协议（诊断 → 内容的转译规则）

> 本文档是 destiny-skill 的"内容化输出"模块——把四乡诊断翻译成可发布的小红书 / 抖音 / 视频号 / 公众号内容。
> **这是诊断工具的应用出口**，不是把诊断降格成营销。
>
> 配合 [`style-anchors.md`](style-anchors.md) 的文风锚点使用。
>
> **cross-ref**：`cheat-on-content@x.y` — 借用其 5 阶段闭环做内容质量校准。

---

## 一句话定位

> **修行做骨，金句做皮。** 诊断的"命运题"是骨架，文风锚点的"咯噔感"是皮肤。两层都不能省——只写骨架是社论文，只写皮肤是营销号。

---

## 核心原则（六条不可破）

1. **必须根植于具体诊断案例** —— 不写"现代人都很焦虑"这种抽象稿。必须从"我身边 / 我自己 / 我看到一个具体人"的一个具体场景切入
2. **不许说教** —— 不用"你应该""你必须""要学会"。所有的"应该"翻译成"我观察到"或"我想试试"
3. **每条内容必须有 1 句点睛** —— 让人"咯噔一下"。可参考 [`style-anchors.md`](style-anchors.md) 的金句样本
4. **严格遵守 destiny-skill 红线** —— 不做操控术、不追击、不冷暴力、不借命运叙事绑架现实反馈
5. **形式逻辑 → 辩证逻辑的微升级** —— 给受众一个"原来还能这么想"的小入口，不强推升级
6. **不留"我写得真好"的回旋镖** —— 内容发出后不再评价自己，留读者空间

## 输入：四乡诊断 → 输出：内容结构

### 输入端（必填）

| 字段 | 来源 | 用途 |
|---|---|---|
| 命运题 | 用户的具体场景 | 内容的核心张力 |
| 住的乡 | 四乡诊断结论 | 决定视角 |
| 思维层次 | 形式/辩证 | 决定语气 |
| 关联案例 | 用户的真实事件或身边观察 | 内容的"故事" |

### 输出结构（小红书/视频号短文版，≤150 字优先）

```text
[开头钩子]（≤15 字，用金句锚点的句式）
[场景描述]（30-50 字，一个具体画面）
[诊断/观察]（50-80 字，不说教，用"我"视角）
[点睛句]（≤25 字，让人心里咯噔一下）
[留白]（让读者自己补完的部分，0-20 字）
```

**字数限制**：150 字以内最佳，最多不超过 250 字。超长就拆成系列。

### 输出结构（视频口播版，60-90 秒）

```text
[冷开场]（≤5 秒，一句话钩子）
[场景]（15-20 秒，一个具体画面 / 一句话场景）
[诊断]（20-30 秒，四乡观察 + 诊断语言）
[余味]（10-15 秒，留白 + 点睛句）
```

## 文风规则（强约束）

### 必须遵守

- **短句优先** —— 一行不超过 25 字
- **断行造呼吸感** —— 一段 1-3 行
- **比喻优先** —— 不用直说
- **用"我"不用"你"** —— 我看到 / 我试过 / 我还在学
- **余味 ≥30%** —— 写完一段，问"读者自己补什么？"——留那部分
- **第一句必须有钩子** —— 否则滑过

### 禁止出现

- ❌ "你应该…" / "你必须…" / "不要…"
- ❌ "作为一个 X" / "现代人都…" / "我们每个人…"
- ❌ "我身边有个朋友..."（伪装第三人称——要写就直接写"我"）
- ❌ 鸡汤式对仗："越 X，越 Y"
- ❌ 营销钩子："建议你…/今天教你一招…"
- ❌ 自夸光环："作为 X 年经验的 X"
- ❌ 命运叙事追击："这是缘分""上天安排""命里注定"

## 诊断→内容的转译规则

### 自大乡 → 内容

**诊断角度**：我执太重 → 看不见别人 → 关系停在"我对/你错"

**内容角度**：
- 写"我曾以为自己对"的瞬间
- 不写"教你放下我执"——写"我试着闭嘴三秒后发现的事"
- 留白方向：让读者自己看见自己也有这刻

**关键词场**：闭嘴 / 三秒 / "原来他没说错" / 我执 / 看见 / 放空

### 幻想乡 → 内容

**诊断角度**：用未来逃避现在 / 画饼不停

**内容角度**：
- 写"我曾经为 X 计划逃了 Y 久"的具体场景
- 不写"别画饼"——写"我把'以后会更好'翻译成下一步动作后发生的事"
- 留白方向：让读者自己问"我现在是不是也在用'以后'骗自己"

**关键词场**：以后 / 翻译 / 一步 / 落地 / 此刻 / 三个月

### 光环乡 → 内容

**诊断角度**：维护人设 / 怕被看穿 / 关系停在表面

**内容角度**：
- 写"我曾经在 X 场合假装 X"的具体尴尬
- 不写"不要做作"——写"那次我没装，他们反而更喜欢我"
- 留白方向：让读者自己识别"我是不是也在维护一个不是我的版本"

**关键词场**：装作 / 标签 / 滤镜 / 看见真的我 / 退后一步 / 不装的瞬间

### 叙事乡 → 内容

**诊断角度**：用付出绑架 / 算账 / 主角是别人的反应

**内容角度**：
- 写"我付出了这么多，他为什么不…"的瞬间
- 不写"别算账"——写"我把付出翻译成'我选'而不是'他欠'后发生的事"
- 留白方向：让读者自己看见"我是不是也在算账"

**关键词场**：我选 / 不是应该 / 我愿意 / 算账 / 故事 / 章节

## 四类典型卡点 → 内容骨架（参考）

### 模式一：付出型 → 写"算账那一刻"的觉醒

```
我曾以为付出就该有回报。
后来发现——
我把"我选了"听成了"我应该"。
一字之差，十年委屈。

（停）
```

### 模式二：讨好型 → 写"压抑自己那一刻"的发现

```
我总怕说错。
于是假装同意，假装高兴，假装喜欢。
直到那天我没装——
他们说："这才是你啊。"

（停）
```

### 模式三：画饼型 → 写"未来翻译成当下"的瞬间

```
我曾经活在"以后就好了"的版本里。
直到那天我把"以后会好"翻译成"这周第一步"——
生活才开始动。

（停）
```

### 模式四：人设型 → 写"装不住那一刻"的释放

```
我花了三年维护一个"懂很多"的人设。
直到那天我说"我不知道"——
空气突然轻了。

（停）
```

## 红线（不发布的内容）

下列模式**直接拒写**，无论用户怎么说：

| 模式 | 为什么拒 |
|---|---|
| 用焦虑、嫉妒、亏欠驱动"你应该抓紧他/她" | 操控术 |
| 把他心通读心化："他其实想的是…"当确定事实 | 越界 |
| 把光环叙事当包装骗术 | 包装 |
| 把距离通当冷暴力建议 | 冷暴力 |
| 把"缘分/命运/修炼"当追击/越界理由 | 命运叙事绑架 |
| 把上帝视角当审判众生 | 傲慢 |
| 把男性价值当索取回应资格证 | 反向物化 |
| 把"遇不到/抓不住"解释成世界/对方亏欠 | 受害者叙事 |

## 与 cheat-on-content 的协作（cross-ref）

如果你同时装了 `cheat-on-content`，本模块可借用其循环：

| 阶段 | cheat-on-content 的对应 | destiny-skill 的输入 |
|---|---|---|
| 选题 | `/cheat-seed` | 命运题 |
| 评分 | `/cheat-score` + `/cheat-score-blind` | 文风锚点匹配度 |
| 预测 | `/cheat-predict` | 互动预判（评论关键词预测） |
| 复盘 | `/cheat-retro` + `## 住相警示` | 是否被住相污染 |
| 进化 | `/cheat-bump` | 文风锚点是否需要更新 |

> ⚠️ **不进 rubric 公式**——文风锚点是写作 lens，不是打分维度。blind 通道隔离继续生效。

## 输出检查清单（每条内容发出前自查）

```text
□ 是否从一个具体场景切入？（不是抽象说理）
□ 是否用"我"不用"你"？
□ 是否有一句点睛？（让人心里咯噔）
□ 是否留 30% 余味给读者？
□ 是否有说教语（"你应该/你必须"）？删除
□ 是否有营销钩子？删除
□ 是否有命运叙事绑架？删除
□ 字数是否 ≤150 字（短文）/ 60-90 秒（口播）？
□ 写完后我还想加东西吗？想加就别加——已经够了
```

## 已知局限

1. **文化语境** —— 当前锚点偏向中文情感类。其他语言/文化需重写锚点
2. **平台差异** —— 小红书 vs 抖音 vs 视频号 的算法偏好不同，本模块只解决"内容质量"层，不解决"分发"
3. **个人差异** —— 文风锚点是参考，不是教条。创作者可基于自己声音调整
4. **金句 ≠ 真理** —— 让人"咯噔"的金句有时只是共振而非真知。要保持"咯噔 ≠ 对"的自觉
